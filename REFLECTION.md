# Reflection on Using AI Agents

Using AI agents in this assignment changed how I approached software work. Instead of writing everything from scratch in a strict linear way, I worked in short cycles: clarify requirement, generate implementation, validate with tests, and refine. The biggest learning for me was that AI is most powerful when I give precise context and constraints. When I asked broad prompts, I often got "close but not exact" outputs. When I gave concrete requirements (expected fields, route names, validation rules, migration constraints), the quality improved a lot. I also learned to treat AI output as a draft, not final truth. Reading generated code carefully, checking type signatures, and verified the behavior against real data was essential.

In terms of efficiency, AI agents clearly saved time compared to fully manual coding. Boilerplate-heavy work (interfaces, service wiring, route handlers, repetitive schema updates, and test scaffolding) was much faster. Refactoring across multiple files was also significantly quicker because the agent could maintain naming and structural consistency. For this project, that translated into faster delivery of features like banking/apply summary flow, pooling validation updates, and migration adjustments. Manual coding would still be possible, but it would likely take longer for the same breadth of changes. The time savings were strongest in repetitive and structural tasks; they were weaker in edge-case-heavy logic where deeper reasoning and verification were required.

At the same time, I noticed limits. AI occasionally produced code that looked correct but did not match the real implementation details (for example, outdated type shapes or simplified migration snippets). This created extra review work. Next time, I would improve the workflow in three ways. First, I would enforce a "verify-before-accept" checklist for every generated block: find real file location, run typecheck, run tests, and compare against expected runtime behavior. Second, I would keep prompts tighter by explicitly specifying data types, exact function signatures, and expected error handling. Third, I would separate "documentation examples" from "exact source code" more clearly, so docs never drift from implementation. Overall, AI agents gave strong productivity gains, but the best results came from combining them with disciplined validation and clear engineering judgment.
